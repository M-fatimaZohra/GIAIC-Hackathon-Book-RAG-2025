# Docusaurus Chapter Creation Skill

## YAML Frontmatter
```yaml
name: docusaurus-chapter-creation-skill
description: A skill for safely adding new chapters and topics to a Docusaurus documentation site following proper structure and conventions
version: 1.0.0
```

## When to Use This Skill

Use this skill when you need to:
- Add a new documentation chapter to a Docusaurus site
- Create new topics within an existing or new chapter
- Ensure proper Docusaurus file structure and navigation integration
- Generate MDX content that follows Docusaurus conventions
- Maintain consistent documentation organization

## Process Steps

1. **Gather User Input**
   - Collect chapter name
   - Collect topic name(s) and summaries
   - Determine desired structure (single file or folder-based)

2. **Validate Input**
   - Check for valid chapter/topic names (no invalid symbols)
   - Ensure proper formatting of inputs
   - Verify structure selection is valid

3. **Determine File Structure**
   - For Option A: Create single MDX file in docs/
   - For Option B: Create folder in docs/ with MDX files inside
   - Sanitize filenames to remove invalid characters

4. **Generate MDX Content**
   - Create proper YAML frontmatter for each topic
   - Generate clean, properly indented MDX content
   - Include appropriate headings hierarchy
   - Format content in beginner-friendly style

5. **Create Content Using Sub-Agent**
   - Call Content Creation Sub-Agent for each topic
   - Pass topic name and summary for expansion
   - Ensure generated content follows MDX compatibility

6. **Update Navigation**
   - Locate existing sidebar configuration
   - Add new chapter/topic to navigation structure
   - Maintain proper ordering with existing documentation

7. **Validation and Output**
   - Verify generated files are syntactically correct
   - Ensure navigation integration is functional
   - Provide final structure and content recommendations

## Output Format

The skill will output:
1. Recommended file structure with exact paths
2. Generated MDX content for each topic
3. Instructions for sidebar/navigation updates
4. Summary of changes needed to complete integration

## Example

### Input
```
Chapter name: "Advanced Components"
Topic names: ["Modal Dialogs", "Data Tables"]
Topic summaries: [
  "Learn how to implement accessible modal dialogs in your React application",
  "Understand how to create responsive data tables with sorting and filtering"
]
Desired structure: Option B (folder-based)
```

### Output
```
Recommended Structure:
docs/advanced-components/
  modal-dialogs.mdx
  data-tables.mdx

Generated Content for modal-dialogs.mdx:
---
title: Modal Dialogs
sidebar_label: Modal Dialogs
---

# Modal Dialogs

[Content generated by sub-agent based on the summary: "Learn how to implement accessible modal dialogs in your React application"]

Sidebar Update:
In sidebars.ts, add to the appropriate category:
{
  type: 'category',
  label: 'Advanced Components',
  items: ['advanced-components/modal-dialogs', 'advanced-components/data-tables'],
}

Files to Create:
- docs/advanced-components/modal-dialogs.mdx
- docs/advanced-components/data-tables.mdx
```
